XGBoosting Class on DataCamp: Chapter 1

Supervised Learning

  • Relies on labeled data
  • You already have some understanding of the past behavior of what you are trying to predict

Supervised learning is always used to solve classification problems with binary results or multi-leveled results.
For example, we can predict whether a borrower will default his loan, this will be a binary classification problem because we either predict a borrower to “yes” or “no”. There could also be a scenario in which we are asked to recognize different kinds of birds on a picture. This will be a classification problem with multi-leveled results.

How to judge the performance of a model?

The most commonly used method is ROC curve and AUC.
The larger the area under the ROC model, the better the model is.

Sometims, Accuracy can be useful:
Accurady = TP+TN/(Total number of predictions made)

Some considerations of supervised learning

  • Features can be categorical or numerical
  • Numeric features should be scaled.
  • Categorical features must be encoded. The most commonly used method is “one-hot” encoding.

What makes XGBoost so popular in the community?

First of all, its calculation speed is very fast. Secondly, it consistently outperforms single-algorithm methods.

Since XGBoost uses Trees as its basic learners, we need to understand the decision Trees.

Decision Tree

Binary Tree is a model that composed of several binary decisions in each node. For each node, there could only be 2 possible results.
The predictions will happen at the leaves of the decision tree.

Individual decision tree is a model tend to have low bias and high variance. That means decision trees are always good at telling the relationship of training data but does poorly if you apply the model to a new dataset. (Overfitting is a big problem)

What is Boosting?

Boosting : it is not a single machine learning models. It is the name of a set of algorithms that follow the same basic ideas.

Here, we need to understand what is Weak learners and Strong learners:
Weak learners are those models who performance are just little better than chance(50%).(存在的意义仅仅比随机碰运气好一点)

XGBoost is a concept that can be used to convert a group of weak learners to a strong learner.

具体措施：

  1. 持续地用一组弱学习器去学习数据集的子集。
  2. 根据每个弱学习器的表现，给每个弱学习器设定一个训练比重（weight)。
  3. 利用每个弱学习器的预测结果，以及每个弱学习器的比重，合成一个总的经过加权的预测结果。
  4. 加权合成之后的预测结果，要好于任何一个弱学习器的预测表现。

When should I use or not use XGBoost?

Use it when :
* You have a large training sample(the # of features < # of training samples )

Do not use it for the following scenarios:
* NLP
* Computer Vision
* Image recognition
(There problems can be better handled with Deep Learning)
